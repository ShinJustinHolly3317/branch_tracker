網域說明

Vercel 預設網址永遠是 *.vercel.app，不是 *.api。專案名稱含 api 只是命名，例如 branch-tracker-api-cxg5.vercel.app 是正確的。

需開通帳號與 Secrets 對照

平台 用途 必填變數 / Secret 設定位置
GitHub 原始碼 + 每日爬蟲 CI DATABASE_URL Repo → Settings → Secrets and variables → Actions
Supabase Postgres 連線字串（建議 Transaction pooler + sslmode=require） Project Settings → Database → Connection string
Vercel Express API DATABASE_URL、選用 API_PG_POOL_MAX=2 Project → Settings → Environment Variables → Production
Netlify React SPA VITE_API_BASE=https://…vercel.app（無尾隨 /） Site → Environment variables（Build）

各平台設定摘要

Vercel（API）

  • Root Directory：apps/api
  • Install（vercel.json）：cd ../.. && npm ci
  • Build：cd ../.. && npm run -w @twbbd/shared build && npm run -w @twbbd/api build
  • 入口：Vercel 偵測 src/index.ts 的 export default app；工廠在 src/createApp.ts
  • 本機 / Docker：改走 src/serve.ts（app.listen）

教訓備忘（別再踩）

  • Vercel ≠ Dockerfile：不能用 apps/api/Dockerfile 直接部署；Docker 留給 compose / Render / Fly。
  • Monorepo：Root = apps/api 時，install/build 必須 cd ../.. && npm ci 並先 build @twbbd/shared。
  • Vercel Express（2024+）：src/app.ts 需 export default Express 實例；勿用過時的 serverless-http + api/index.ts 雙入口。
  • Ready ≠ 能用：Build 綠燈只代表編譯過；要看 error rate 與 /health。
  • VITE_* 是 build time：改 Netlify API 網址後一定要重新 deploy 前端。
  • ingest 與 API 共用 DATABASE_URL，但 GitHub Secrets 與 Vercel Env 要分別設定。
